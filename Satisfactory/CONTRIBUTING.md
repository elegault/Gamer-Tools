# Contributing

Thanks for helping improve Satisfactory Train Mapper.

## Setup

1. Open the repository in VS Code or Codespaces.
2. Install dependencies in `Satisfactory/app` with `npm ci`.
3. Run `npm run dev` for local work.
4. Run `npm run build` before opening a pull request.

## Pull Requests

- Keep changes focused on one behavior or feature when possible.
- Describe any map schema or persistence changes in the PR body.
- Include screenshots for visible UI changes.
- Mention any follow-up migration concerns if you change saved-map data.

## Code Style

- Prefer small, local edits that preserve existing patterns.
- Keep schema defaults and store logic aligned.
- Update documentation when you add or change user-facing behavior.
